# Contributing to the EAAP Protocol

Thank you for your interest in contributing to the Exogram Action Admissibility Protocol.

## How to Contribute

### Architectural Feedback

If you have questions, concerns, or proposals about the protocol's architecture, open a [GitHub Issue](https://github.com/Richard-Ewing/exogram-protocol-rfc/issues) with the **[Architecture]** prefix.

Examples of valuable architectural feedback:
- "Layer 3 should also consider X when evaluating admissibility"
- "The state hash formula doesn't account for Y"
- "How does EAAP handle Z edge case?"

### Schema Corrections

If you find errors in data payloads, API contracts, or response codes, submit a [Pull Request](https://github.com/Richard-Ewing/exogram-protocol-rfc/pulls) with the correction.

### Security Reviews

If you identify a security vulnerability in the protocol specification, please see [SECURITY.md](SECURITY.md) for responsible disclosure.

## What We're Looking For

- **Technical scrutiny** — We want the protocol to be torn apart and rebuilt stronger
- **Edge case identification** — Adversarial conditions we haven't considered
- **Integration patterns** — How EAAP fits into frameworks not yet listed
- **Conformance testing** — Approaches to verifying EAAP compliance

## What This Repository Is NOT

This repository contains the **protocol specification only**. It does not contain:
- Exogram's proprietary backend implementation
- Production infrastructure code
- API keys, secrets, or configuration

## Code of Conduct

Be constructive. Be specific. Be technical. We welcome disagreement — but it must be backed by reasoning, not opinion.
