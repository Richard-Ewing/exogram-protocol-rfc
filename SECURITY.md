# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the EAAP protocol specification, **please do not open a public issue**.

Instead, report it privately via email:

**richard@exogram.ai**

Include:
- Description of the vulnerability
- Steps to reproduce or proof of concept
- Potential impact assessment
- Suggested mitigation (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 7 business days
- **Resolution**: Dependent on severity; critical issues addressed immediately

## Scope

This security policy covers the **protocol specification** (this repository). For vulnerabilities in the production Exogram.ai service, contact richard@exogram.ai with subject line "Security: Production".

## Recognition

Security researchers who responsibly disclose valid vulnerabilities will be credited in the RFC changelog (with permission).
